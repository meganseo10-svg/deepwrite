# 06. UI SPEC — DEEPWRITE 화면

> deepread 디자인 토큰 그대로. 모바일 우선. 다크모드 지원.

## 화면 1: 작문 에디터 `/write` (핵심)
**레이아웃: 좌(입력) / 우(분석) 2단. 모바일은 탭 전환.**

좌측 입력 영역
- 상단: **톤 선택 세그먼트** `[격식체] [중립] [구어]` (선택값이 강조 표시)
- 그 아래: **힌트 모드 토글** `즉시 / 시도 후 / 끄기`
- 본문 textarea (자동 높이)
- 작성 중 동작:
  - 단어 입력 → (힌트모드에 따라) **콜로케이션 힌트 팝오버** (하단 떠오름)
  - 단어 더블클릭/선택 → **비교 단어 카드** 팝오버 (Pro 게이트 시 자물쇠)
- 하단: `[ 진단받기 ]` 버튼

우측 분석 영역 (진단 후)
- **5차원 점수**: 5개 막대(0~100) + 총평 한 줄
- **리라이트 Diff**: 원문↔리라이트 단어 단위 색상(초록 추가/빨강 삭제/노랑 교체)
- **"왜 이 표현인가" 리스트**: 각 변경 카드 = before→after + 이유 + 규칙 + 빈도배지
- 하단: `[ 3톤으로 비교 ]` `[ 표현 저장 ]` `[ 다시 쓰기(개작) ]`

## 화면 2: 3톤 비교 (모달 또는 `/write` 하단 확장)
- 3컬럼: 격식체 / 중립 / 구어 (각 카드)
- 아래 **드라이버 표**: 동사·연결어·명사화·신호어가 톤별로 어떻게 다른지
- `consistency_tip` 안내 배너

## 화면 3: 역번역 `/backtranslate`
- 상단: 제시된 **한국어 의도** 카드
- 가운데: 영어 입력 textarea → `[ 제출 ]`
- 결과:
  - **역번역된 한국어**(내 영어를 직역) 카드
  - **의도 일치도** 큰 숫자(0~100) + 게이지
  - **소실된 뉘앙스** 리스트(intended → lost)
  - **네이티브 모범** 카드
- `[ 새 문제 ]`

## 화면 4: 약점 리포트 `/weakness`
- 오류 유형 **랭킹 막대**(관사/콜로케이션/시제/전치사/run-on…) + 횟수
- 각 항목 **추이 배지**(지난주 대비 ±%)
- **오늘의 맞춤 드릴** 카드 → 클릭 시 해당 유형 집중 문제

## 화면 5: 대시보드 `/dashboard`
- 오늘의 과제(약점 맞춤) / streak / 5차원 점수 추이 미니차트 / 최근 작문
- 헤더에 **"← DEEPREAD로"** 딥링크

## 화면 6: 온보딩 `/onboarding`
- 짧은 작문 3개(장르·톤 안내) → 진단 결과(추정 CEFR + 최약점) → 대시보드 진입
